## 4. 지역별 확장 방식

- **현재**  
  단지 제휴 페이지에는 “지역” 필드나 지역별 안내 문구가 없습니다. 문의는 카톡·전화로만 받도록 되어 있어, 모든 지역을 동일하게 안내할 수 있습니다.

- **지역별로 나누고 싶을 때**  
  1. **옵션 A: 문구만 분리**  
     - `app/complex/page.tsx` 안에 “가능 지역: 경기·서울·인천…” 같은 문단을 두고, 필요 시 `process.env.NEXT_PUBLIC_COMPLEX_REGIONS` 등으로 치환.  
  2. **옵션 B: 지역별 라우트**  
     - 예: `/complex?region=gyeonggi`, `/complex/gyeonggi`  
     - 해당 경로용 레이아웃/페이지에서 지역명·연락처(선택)만 바꾸고, 나머지 섹션은 공통 컴포넌트로 재사용.  
  3. **옵션 C: 동일 페이지 + FAQ**  
     - “어느 지역부터 가능한가요?” FAQ에 “현재 ○○ 지역 중심으로 진행 중이며, 타 지역은 문의 주시면 안내드립니다” 등으로 정리.

지역 제한이 생기면 C로 FAQ만 추가해도 되고, 지역별 랜딩이 필요하면 B처럼 경로를 나누면 됩니다.
