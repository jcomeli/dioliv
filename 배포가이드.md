# Vercel 배포 + 도메인 추가 가이드

아래 순서대로만 하시면 됩니다. 도메인은 나중에 추가해도 됩니다.

---

## 1단계: GitHub에 코드 올리기

Vercel은 GitHub 저장소를 연결해서 배포합니다. 아직 GitHub에 안 올렸다면:

1. **GitHub 가입**  
   - [github.com](https://github.com) 접속 → Sign up (이메일로 가입)

2. **새 저장소(Repository) 만들기**  
   - 로그인 후 오른쪽 위 **+** → **New repository**  
   - Repository name: `window-cleaning` (원하는 이름으로 해도 됨)  
   - **Public** 선택  
   - **Create repository** 클릭

3. **지금 프로젝트 폴더를 GitHub에 올리기**  
   - 터미널(또는 VS Code 터미널)을 열고, **프로젝트 폴더 안에서** 아래를 **한 줄씩** 실행하세요.

   ```bash
   git init
   git add .
   git commit -m "첫 배포"
   git branch -M main
   git remote add origin https://github.com/여기에본인아이디/저장소이름.git
   git push -u origin main
   ```

   - `여기에본인아이디` → 본인 GitHub 아이디  
   - `저장소이름` → 2번에서 만든 이름(예: window-cleaning)  
   - 예: `https://github.com/hong/window-cleaning.git`

   - `git push` 할 때 GitHub 로그인 창이 뜨면 로그인하거나, **Personal Access Token**을 비밀번호처럼 입력하면 됩니다.

---

## 2단계: Vercel 가입 및 GitHub 연결

1. **Vercel 접속**  
   - [vercel.com](https://vercel.com) 접속

2. **가입**  
   - **Continue with GitHub** 클릭  
   - GitHub 로그인/권한 허용하면 Vercel 계정이 만들어집니다.

3. **새 프로젝트 만들기**  
   - **Add New…** → **Project**  
   - **Import Git Repository**에서 방금 올린 저장소(예: window-cleaning) 선택  
   - **Import** 클릭

4. **설정 확인 후 배포**  
   - **Framework Preset**: Next.js 로 되어 있으면 그대로 두세요.  
   - **Root Directory**: 비워 두세요.  
   - **Environment Variables**: 여기서는 비워 두고 **Deploy** 눌러도 됩니다. (도메인 추가할 때 변수 넣으면 됩니다.)

5. **배포 완료**  
   - 1~2분 정도 지나면 **Congratulations** 화면이 나옵니다.  
   - **Visit** 버튼을 누르면 `https://프로젝트이름-xxxx.vercel.app` 같은 주소로 사이트가 열립니다.  
   - 이 주소가 **임시 도메인**입니다. 이걸로 먼저 사이트가 잘 뜨는지 확인하세요.

---

## 3단계: (나중에) 도메인 추가하기

도메인을 살 준비가 되면:

1. **도메인 구매**  
   - 가비아, 카페24, Namecheap 등에서 원하는 주소 구매  
   - 예: `dio-window.com`

2. **Vercel에서 도메인 연결**  
   - [vercel.com](https://vercel.com) 로그인  
   - 해당 프로젝트 클릭  
   - 상단 **Settings** → 왼쪽 메뉴 **Domains**  
   - **Add**에 구매한 도메인 입력 (예: `dio-window.com`)  
   - **Add** 클릭

3. **DNS 설정 (도메인 판매처에서)**  
   - Vercel이 안내하는 대로 **네임서버** 또는 **A 레코드/CNAME**을 도메인 판매처(가비아 등)에서 설정합니다.  
   - 보통 24~48시간 안에 연결됩니다.

4. **사이트 주소를 도메인으로 고정**  
   - Vercel **프로젝트** → **Settings** → **Environment Variables**  
   - **Key**: `NEXT_PUBLIC_SITE_URL`  
   - **Value**: `https://연결한도메인.com` (예: `https://dio-window.com`)  
   - **Save** 후, **Deployments** 탭에서 맨 위 배포 옆 **⋯** → **Redeploy** 한 번 해주세요.  
   - 이렇게 하면 검색/공유 시 항상 도메인 주소가 사용됩니다.

---

## 체크리스트

- [ ] GitHub에 저장소 만들고 코드 push  
- [ ] Vercel 가입 후 GitHub로 로그인  
- [ ] Vercel에서 해당 저장소 Import → Deploy  
- [ ] `https://xxx.vercel.app` 로 사이트 열리는지 확인  
- [ ] (도메인 있을 때) Vercel Domains에 도메인 추가  
- [ ] (도메인 있을 때) DNS 설정  
- [ ] (도메인 있을 때) `NEXT_PUBLIC_SITE_URL` 환경 변수 추가 후 Redeploy  

도메인 없이 2단계까지 하면 이미 전 세계에서 접속 가능한 주소가 생깁니다. 도메인은 준비되면 3단계만 따라 하시면 됩니다.
